# Настройка GitHub Pages для EDA Dashboard

## Что такое GitHub Pages?

GitHub Pages — это бесплатный сервис для размещения статических веб-сайтов непосредственно из репозитория GitHub. Он автоматически публикует HTML, CSS и JavaScript файлы, что делает его идеальным решением для публикации интерактивных дашбордов и отчетов.

## Структура проекта для GitHub Pages

Проект настроен для работы с GitHub Pages следующим образом:

```
docs/
├── index.html          # Главный файл дашборда (автоматически открывается на GitHub Pages)
├── data/
│   └── eda_data.json   # Данные для визуализации
└── README.md           # Инструкции по настройке
```

## Быстрая настройка (3 шага)

### Шаг 1: Убедитесь, что файлы на месте

Проверьте, что в папке `docs/` есть:
- ✅ `index.html`
- ✅ `data/eda_data.json`

### Шаг 2: Настройте GitHub Pages

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** (настройки)
3. Прокрутите вниз до раздела **Pages** (в левом меню)
4. В разделе **Source**:
   - Выберите **Branch**: `main` (или ваша основная ветка)
   - Выберите **Folder**: `/docs`
5. Нажмите **Save**

### Шаг 3: Дождитесь публикации

- GitHub автоматически опубликует сайт через 1-2 минуты
- Ссылка будет отображаться в разделе Pages: `https://[ваш-username].github.io/[название-репозитория]/`
- Первая публикация может занять до 10 минут

## Обновление данных

Когда вы обновляете данные EDA:

```bash
# 1. Сгенерируйте новые данные
python scripts/eda_analysis.py

# 2. Скопируйте обновленный файл
copy reports\data\eda_data.json docs\data\eda_data.json

# 3. Закоммитьте и запушьте
git add docs/data/eda_data.json
git commit -m "Update EDA dashboard data"
git push
```

GitHub Pages автоматически обновится после пуша (обычно в течение 1-2 минут).

## Локальный просмотр

Для тестирования дашборда локально:

```bash
cd docs
python -m http.server 8000
```

Затем откройте в браузере: `http://localhost:8000`

## Важные замечания

- GitHub Pages поддерживает только **статические файлы** (HTML, CSS, JavaScript)
- Все внешние ресурсы (CDN, изображения) должны быть доступны через интернет
- Размер репозитория ограничен, но для JSON данных это обычно не проблема
- Изменения публикуются автоматически при каждом пуше в основную ветку

## Устранение проблем

**Дашборд не загружается:**
- Проверьте, что файл `docs/data/eda_data.json` существует
- Убедитесь, что путь в HTML правильный: `data/eda_data.json`
- Проверьте консоль браузера (F12) на наличие ошибок

**Данные не обновляются:**
- Убедитесь, что вы скопировали файл в `docs/data/`
- Проверьте, что изменения закоммичены и запушены
- Подождите несколько минут для обновления GitHub Pages
